Предыстория.

Система контроля версиями в своем упрощенном виде представляет из себя структуру данных, в которых сохранены все предыдущие состояния редактируемого документа. В своем локальном виде это каталок из папок, с сохраненными на каждый момент времени рабочими файлами, который расположен на одном рабочем месте.

Далее, в централизованных системах подявилась возможность работать совместно, при этом синхронизируя с центральным сервером основной рабочий проект или документ, что позволяло выполнять работу с большим количеством людей. При этом сам проект был подвержен угрозе выхода из строя самого сервера и полной потери всех дынных с внесеными изменениями и наработками каждого сотрудника.

Лишеной всех недостатков является распределенная система конроля версий. Где на каждый момент времени у всех участниках проекта абсолютно идентичная копия всех данных проета, т.е. весь его репозиторий. В случае утери одной из копий ее всегда можно восстановить с локальной версии любого участника.

Git является инструментом, позволяющим реализовать распределенную систему контроля версиями, а GitHub - сервис для проектов, использующих Git.

Руководство Git по основным операциям

Разные команды командной строки

  • alias iy='I am go to scool today' - Создать сокращенную команду "iy", вызов которой в командной строке: "git iy" выполнит полную команду: "git I am go to scool today".
  • git config --global --unset alias.iy - Удалить сокращенную команду "iy".
  • git config --global --edit - Редактировать файл конфигурации git.
  • git tag {v1} - Назначить тэг текущей версии коммита, где {v1} - имя тэга.
  • git checkout {v1} - Переход к коммиту с тэгом {v1}.
  • git checkout {v1}^ - Переход к предыдущему коммиту тэга {v1}.
  • git tag - Просмотр всех тэгов.
  • git checkout master - Вернуться на последнюю рабочую ветку.
  • git checkout {hash} - Где {hash} это 7 цифр кода состояния изменений, полученные в результате команды "git log".
  • git fetch - Скачать объекты и ссылки с другого репозитория.
  • git merge - Объединение одной или нескольких историй разработки вместе.
  • git pull - Извлечение изменений и объединение с другим репозиторием или локальной веткой, аналог команд fetch и merge.
  • git push - Обновление внешних ссылок и связанных объектов.
  • git remote add origin {https://github.com/VadMih/test2.git} - Добавит удаленный репозиторий.
  • git remote remove https://github.com/VadMih/testing.git - Удалить удаленную репозиторий.
