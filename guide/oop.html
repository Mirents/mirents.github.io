Базовые принципы ООП в Java

Определение.

Объектно-ориентированное программирование (ООП) - методология программирования, основанная на представлении программы в виде совокупности объектов, каждый из которых является экземпляром определенного класса, а классы образуют иерархию наследования. ООП базируется на четырех основных принципах, являющихся основополагающими.

Наследование.

Принцип наследования реализует создание объекта класса-потомка с методами и атрибутами класса-родителя, но с возможностью расширения дополнительной, индивидуальной функциональностью.

Абстракция.

Абстракция как принцип в ООП реализует описание общих классов, в которых указаны обобщенные методы и атрибуты, в последствии при необходимости реализуемые в классах-наследниках.

Инкапсуляция.

Принцип инкапсуляции реализует ограниечение и полный запрет на изменение данных внутри класса-родителя с помощью модификаторов доступа, а так же методов геттеров и сеттеров.

Полиморфизм.

Принцип полиморфизма выражает возможность работать с несколькими классами-потомками определенного класса-родителя будто это один тип. При этом в каждом конкретном случае поведение будет различным, в зависимости от типа, к которому они принадлежат. Так же при передаче в качестве параметра класса-родителя в метод его потомков, будет вызван соответсвующий классу-потомку метод.

Отличие абстракции и интерфейса.

Абстрактный класс есть некое обобщение, описывающее общий для данного вида класс, у которого не может быть реализации в виде конкретного объекта. При этом он, как и интерфейс не может наследоваться. Абстрактный класс является шаблоном для классов, которые станут родительскими для своих потомков.

Интерфейс в чистом виде есть описание определенного функционала для объектов, даже не связанных между собой, но при этом которым присуща определенное поведение, описываемое вышеуказанным интерфейсом.

Классы могут реализовывать множество интерфейсов, но наследоваться только от одного класса.
